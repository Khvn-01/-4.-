# Система оповещений о доступности сетевых устройств

## 📋 Описание проекта

Автоматизированный сервис для мониторинга доступности сетевых устройств с:
- ✅ Проверкой доступности по ping каждые 5 минут
- 📝 Логированием всех событий в файл `alerts.log`
- 📱 Отправкой уведомлений в Telegram при недоступности устройства
- 🖥️ Выводом информации в консоль в реальном времени

## 🔧 Требования

- Python 3.7+
- Доступ в интернет (для отправки уведомлений в Telegram)
- Библиотека `requests` для работы с API Telegram

## 📦 Установка зависимостей

```bash
pip install requests
```

Или через файл requirements.txt:

```bash
pip install -r requirements.txt
```

## ⚙️ Настройка Telegram-бота

### Шаг 1: Создание бота
1. Откройте Telegram и найдите контакт **@BotFather**
2. Отправьте команду `/newbot`
3. Введите имя бота (например, "NetAlertBot")
4. Введите имя пользователя бота (например, "net_alert_bot_12345")
5. **Сохраните полученный TOKEN** (выглядит как `123456:ABC-xyz123abc`)

### Шаг 2: Узнание своего Chat ID
1. Найдите контакт **@userinfobot**
2. Отправьте ему любое сообщение
3. Бот вернёт ваш **Chat ID** (например, 987654321)

### Шаг 3: Внесение данных в скрипт
Отредактируйте файл `net_alert.py` и найдите строки:

```python
TOKEN = "ВАШ_TELEGRAM_TOKEN"  # Вставьте сюда токен
CHAT_ID = "ВАШ_CHAT_ID"        # Вставьте сюда Chat ID
```

Замените их на ваши реальные значения:

```python
TOKEN = "123456:ABC-xyz123abc"
CHAT_ID = "987654321"
```

## 🔍 Основные функции скрипта

### `ping(ip, count=1, timeout=2)`
Проверяет доступность хоста по ping
- **Параметры:** IP-адрес, количество пакетов, таймаут
- **Возвращает:** True если доступен, False если недоступен

### `send_telegram(message)`
Отправляет сообщение в Telegram
- **Параметры:** Текст сообщения
- **Возвращает:** True если отправлено успешно

### `load_devices()`
Загружает список устройств из devices.json
- **Возвращает:** Список словарей с устройствами

### `check_devices(devices)`
Проверяет доступность всех устройств и отправляет уведомления
- **Параметры:** Список устройств

## 💡 Советы и рекомендации

1. **Тестирование без Telegram:**
   - Временно установите неверные TOKEN и CHAT_ID
   - Программа будет работать, но без отправки уведомлений

2. **Добавление новых устройств:**
   - Просто добавьте строку в devices.json
   - Перезагрузите скрипт

3. **Изменение интервала проверки:**
   - Измените `CHECK_INTERVAL = 300` на нужное значение (в секундах)
   - 300 секунд = 5 минут

4. **Запуск в фоновом режиме (Windows):**
   ```bash
   pythonw.exe net_alert.py
   ```

5. **Запуск как служба (Windows):**
   - Используйте NSSM или встроенный Планировщик задач

## 🐛 Решение проблем

### "ModuleNotFoundError: No module named 'requests'"
```bash
pip install requests
```

### Telegram не отправляет сообщения
- ✅ Проверьте интернет-соединение
- ✅ Убедитесь, что TOKEN и CHAT_ID правильные
- ✅ Убедитесь, что бот не был заблокирован

### ping не работает
- На некоторых системах требуются права администратора
- Запустите консоль от администратора

### Ошибка "Connection refused" в Telegram API
- Может быть блокировка API Telegram в вашей стране
- Используйте VPN
